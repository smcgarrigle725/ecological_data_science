# Data

Input datasets are **not included** in this repository. This file describes the required datasets, their column definitions, and how to obtain or prepare them.

CSV files should be placed in this directory before running the example pipeline notebook or any database upload scripts.

---

## Required Files

| File | Rows | Description |
|------|------|-------------|
| `Site.csv` | One per site | Sampling location metadata |
| `Animal.csv` | One per animal sample | Bivalve/animal microparticle data |
| `Sediment.csv` | One per sediment sample | Sediment microparticle data |
| `Water.csv` | One per water sample | Water microparticle data |
| `FTIR.csv` | One per MP particle | Micro-FTIR spectroscopy results |

---

## Column Definitions

### Site.csv

| Column | Type | Description |
|--------|------|-------------|
| `siteid` | string | Autogenerated: Year + Site + sample type (e.g. `2022_HarbourNorth_Animal`) |
| `collection_organization` | string | Name of organization collecting the samples |
| `primarycontact` | string | Contact person at the organization |
| `year` | string | Year samples collected |
| `waterbody` | string | Name of waterbody |
| `waterbodytype` | string | `Freshwater`, `Marine`, or `Brackish` |
| `site` | string | Name of site |
| `latitude` | float | Decimal degrees |
| `longitude` | float | Decimal degrees |
| `substrate` | string | Description of substrate at the site |
| `benthicpelagic` | string | `Benthic` or `Pelagic` |
| `tidaldepth` | string | e.g. `intertidal`, `subtidal` (marine only) |
| `samplestaken` | integer | Number of samples taken |
| `sampletypes` | string | `Animal`, `Sediment`, or `Water` |

## Notes

- `siteid` format: `Year_Site_SampleType` — used as the join key between Site and sample tables.
- The FTIR table does not include a `sample_type` column. The originating sample matrix (`Animal`, `Water`, or `Sediment`) can be inferred from the structure of the `id` string.
- Empty cells in CSVs are handled during upload — `table_upload.py` drops blank values rather than storing empty strings, which DynamoDB does not accept.
- Export CSVs from Excel as **CSV UTF-8** to avoid encoding issues with the byte-order mark (BOM).
